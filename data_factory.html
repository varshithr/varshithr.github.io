Azure Data Factory: Cloud-Scale Data Integration

A deep dive into Azure Data Factory (ADF), the cloud-based ETL and data integration service for orchestrating and automating data movement and transformation.

Core Components

  • Pipelines: A logical grouping of activities that together perform a task. A pipeline could ingest data, transform it, and then load it into a data store.
  • Activities: Represent a single processing step in a pipeline. There are three types: data movement (e.g., Copy activity), data transformation (e.g., Mapping Data Flows, Spark), and control activities (e.g., ForEach, If Condition).
  • Datasets: Named views of data that simply point to or reference the data you want to use in your activities as inputs or outputs.
  • Linked Services: The connection strings that define the connection information needed for Data Factory to connect to external resources (e.g., Azure Blob Storage, an on-premises SQL Server).
  • Triggers: The unit of processing that determines when a pipeline execution needs to be kicked off. Triggers can be scheduled, tumbling window, storage event-based, or custom event-based.
  • Integration Runtimes (IR): The compute infrastructure used by Data Factory to provide data integration capabilities across different network environments. There are three types: Azure, Self-Hosted, and Azure-SSIS.

Key Features

  • Code-Free and Code-First: Provides a rich visual interface for building pipelines with no code required, but also allows for advanced scripting and integration with SDKs.
  • Mapping Data Flows: A powerful, visual data transformation experience that generates and executes Spark code on a managed Spark cluster, allowing you to build complex ETL logic without writing code.
  • Hybrid Data Integration: The Self-Hosted Integration Runtime allows you to securely move data between the cloud and on-premises data stores.
  • Extensive Connectivity: Supports over 90 built-in connectors to a wide variety of SaaS, on-premises, and cloud data sources.
  • CI/CD Integration: Natively integrates with Azure DevOps and GitHub for source control and automated deployment of pipelines.

Data Factory vs. Synapse Pipelines

Azure Synapse Analytics includes a feature called Synapse Pipelines, which is based on the same technology as Azure Data Factory. However, there are some key differences:

  • Scope: Data Factory is a standalone data integration service. Synapse Pipelines are one component of the broader Synapse Analytics workspace, which also includes Spark pools, SQL pools, and Data Explorer.
  • Integration: Synapse Pipelines are more tightly integrated with the other compute engines within the Synapse workspace.
  • User Experience: While the core functionality is the same, the user experience is slightly different, with Synapse providing a more unified studio for all analytics tasks. For pure data integration and orchestration, Data Factory is often the preferred choice. For building an end-to-end analytics platform, Synapse provides a more seamless experience.
