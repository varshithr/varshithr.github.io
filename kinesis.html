Amazon Kinesis: Real-Time Data Streaming

A deep dive into the Amazon Kinesis family of services for collecting, processing, and analyzing streaming data at any scale.

The Kinesis Family

Kinesis is not a single service, but a suite of services designed for different streaming data use cases.

  • Kinesis Data Streams: A massively scalable and durable real-time data streaming service. It can continuously capture gigabytes of data per second from hundreds of thousands of sources. You manage the capacity by provisioning "shards."
  • Kinesis Data Firehose: The easiest way to reliably load streaming data into data lakes, data stores, and analytics services. It's a fully managed service that automatically scales to match the throughput of your data.
  • Kinesis Data Analytics: Process and analyze streaming data in real time with SQL or Apache Flink. It allows you to build complex streaming applications to perform time-series analytics, feed real-time dashboards, and create real-time alerts.
  • Kinesis Video Streams: Securely stream video from connected devices to AWS for analytics, machine learning, and other processing.

Kinesis Data Streams: Core Concepts

  • Stream: An ordered sequence of data records. A stream is composed of one or more shards.
  • Shard: A uniquely identified sequence of data records in a stream. Each shard has a fixed capacity (1 MB/s and 1,000 records/s for writes; 2 MB/s for reads). The total capacity of a stream is the sum of the capacities of its shards.
  • Partition Key: Used to group data by shard within a stream. Kinesis uses the partition key to segregate and route records to different shards. Records with the same partition key always go to the same shard.
  • Producers and Consumers: Producers put data records into Kinesis Data Streams. Consumers get data records from Kinesis Data Streams. Consumers can be custom applications (using the KCL), Lambda functions, or other AWS services like Kinesis Data Firehose or Kinesis Data Analytics.

Common Architectural Patterns

  • Real-Time Analytics: `Producers -> Kinesis Data Streams -> Kinesis Data Analytics (Flink/SQL) -> Real-time Dashboards / Alerts`
  • Stream Archiving: `Producers -> Kinesis Data Streams -> Kinesis Data Firehose -> Amazon S3`
  • Serverless Stream Processing: `Producers -> Kinesis Data Streams -> AWS Lambda -> DynamoDB / Other AWS Services`
