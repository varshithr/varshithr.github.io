Azure Event Hubs: Big Data Streaming Platform

A deep dive into Azure Event Hubs, the fully managed, real-time data ingestion service that's simple, trusted, and scalable.

Core Concepts

  • Event Hubs Namespace: A container for one or more Event Hubs.
  • Event Hub: A partitioned data stream, analogous to a topic in Kafka or a stream in Kinesis.
  • Partitions: An ordered sequence of events that is held in an Event Hub. Partitions are a mechanism for data parallelism.
  • Event Publishers (Producers): Any entity that sends data to an Event Hub.
  • Event Receivers (Consumers): Any entity that reads event data from an Event Hub.
  • Consumer Groups: A view (state, position, or offset) of an entire event hub. Consumer groups enable multiple consuming applications to each have a separate view of the event stream.
  • Throughput Units (Standard) / Processing Units (Premium): The pre-purchased units of capacity that control the throughput of your Event Hub.

Key Features

  • Scalability: Capable of ingesting millions of events per second, making it suitable for the most demanding big data workloads.
  • Protocol Support: Supports multiple protocols, including AMQP, Kafka, and HTTPS.
  • Capture: Automatically capture streaming data in Event Hubs and save it to Azure Blob Storage or Azure Data Lake Storage.
  • Geo-disaster recovery: Provides geographic disaster recovery and geo-replication capabilities.

Event Hubs vs. IoT Hub

  • Scope: Event Hubs is a general-purpose event ingestion service. IoT Hub is a specialized service for connecting and managing IoT devices, with features like device twins, per-device identity, and command-and-control capabilities.
  • Directionality: Event Hubs is primarily for data ingestion (one-way). IoT Hub is bi-directional, allowing you to send commands back to devices.
  • Use Case: Use IoT Hub when you need to manage and communicate with individual IoT devices. Use Event Hubs for high-throughput data streaming from any source, not just IoT devices.
